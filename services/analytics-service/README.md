# Analytics Service

## 1. Название и назначение сервиса

`analytics-service` — микросервис аналитики в системе опросов.

На текущем этапе сервис отвечает за базовую аналитику по опросам: по идентификатору опроса он возвращает количество полученных ответов. В общей микросервисной архитектуре сервис выступает как отдельный слой аналитики и предоставляет агрегированные данные через собственное HTTP API.

Основные функции:

- отдача базовой статистики по опросу;
- проверка доступности сервиса через health-check;
- обработка ошибок при обращении к связанному микросервису.

## 2. Архитектура и зависимости

### Технологии и фреймворки

- Python 3.11+
- FastAPI
- Uvicorn
- Pydantic v2
- httpx
- pytest

### Взаимодействие с другими микросервисами

В текущей реализации `analytics-service` получает данные из сервиса опросов. Для расчёта базовой статистики сервис делает запрос:

`GET {SURVEY_SERVICE_URL}/surveys/{survey_id}/answers/count`

Ожидается, что сервис опросов вернёт JSON с полем `answers_count`.

После этого `analytics-service` публикует результат через собственный API:

`GET /analytics/surveys/{survey_id}/basic`

### Внешние сервисы

На текущем этапе внешние инфраструктурные сервисы не используются:

- Kafka не используется;
- Redis не используется;
- S3 не используется;
- отдельная база данных в этом микросервисе не подключена.

## 3. Способы запуска сервиса

### Вариант 1. Локальный запуск без Docker

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
$env:SURVEY_SERVICE_URL="http://localhost:8001"
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

После запуска сервис будет доступен по адресу:

`http://localhost:8000`

Для разработки, тестов и запуска линтера удобнее установить dev-зависимости:

```powershell
pip install -r requirements-dev.txt
```

### Вариант 2. Запуск через Docker

```powershell
docker build -t analytics-service .
docker run --rm -p 8000:8000 -e SURVEY_SERVICE_URL=http://host.docker.internal:8001 analytics-service
```

### Переменные окружения

| Переменная | Значение по умолчанию | Назначение |
| --- | --- | --- |
| `SURVEY_SERVICE_URL` | `http://localhost:8001` | Базовый URL сервиса опросов, из которого запрашивается количество ответов |

Примечание: приложение читает переменные окружения напрямую через `os.getenv`. Если вы хотите использовать файл `.env`, его значения нужно передавать через оболочку, Docker или `docker-compose`.

## 4. API документация

FastAPI автоматически публикует документацию:

- Swagger UI: `http://localhost:8000/docs`
- ReDoc: `http://localhost:8000/redoc`
- OpenAPI JSON: `http://localhost:8000/openapi.json`

Основные эндпоинты:

- `GET /health` — проверка, что сервис запущен и отвечает;
- `GET /analytics/surveys/{id}/basic` — возвращает базовую статистику по опросу.

Пример успешного ответа:

```json
{
  "survey_id": 1,
  "answers_count": 7
}
```

Возможные ошибки:

- `404 Not Found` — опрос не найден в связанном сервисе;
- `502 Bad Gateway` — сервис опросов недоступен или вернул ошибку.

## 5. Как тестировать

Запуск тестов:

```powershell
pip install -r requirements-dev.txt
pytest
```

Что покрыто тестами:

- успешное получение базовой статистики;
- корректная обработка `404 Not Found`;
- корректная обработка `502 Bad Gateway`.

Проверка качества кода:

```powershell
ruff check .
ruff format .
```
